# Images in tests/qr_codes_difficulty/ that flatcv currently mishandles.
# Two failure modes are tracked:
#   - positive miss:  `NNNN_text.png` — flatcv fails to decode the QR.
#   - false positive: `noqr-NNNN.png` — flatcv hallucinates a QR in an
#                     image that contains no QR at all.
# Each line is a bug or missing capability in flatcv's QR pipeline that we
# want to fix.
#
# Format: one path per line, relative to tests/qr_codes_difficulty/.
# Lines starting with `#` and blank lines are ignored.
#
# Remove a line when flatcv learns to decode that image — the test will
# flag it as an unexpectedly-fixed entry so this file stays in sync.

# --- Positive misses -------------------------------------------------------

# collapse of the pastel palette leaves the QR nearly invisible and the
# checker fires false finder ratios; neither polarity retry fixes it.
